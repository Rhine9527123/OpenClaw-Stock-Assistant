# 系统架构文档

## 概述

QuantMorning 是一个基于 OpenClaw 的股票投资助手系统，采用模块化设计，支持多数据源、自动故障转移和定时任务调度。

## 核心模块

### 1. 数据收集模块 (Data Collector)

**职责**: 从多个数据源获取股票实时数据

**数据源优先级**:
1. 东方财富 API (主数据源)
2. 腾讯财经 API (备用数据源)
3. 本地缓存 (离线模式)

**故障转移策略**:
```python
# 伪代码
def get_stock_data(code):
    try:
        return eastmoney_api.get(code)
    except:
        try:
            return tencent_api.get(code)
        except:
            return cache.get(code)
```

### 2. 图表生成模块 (Chart Generator)

**职责**: 生成可视化图表

**支持的图表类型**:
- 价格走势图 (含 MA5/MA10/MA20 均线)
- 涨跌幅柱状图
- 成交量分析图
- 技术指标面板

**技术栈**:
- matplotlib - 基础绘图
- pandas - 数据处理
- 中文字体支持 (SimHei)

### 3. 报告生成模块 (Report Generator)

**职责**: 整合数据生成结构化报告

**报告内容**:
- 行情概览 (价格、涨跌幅、成交量)
- 技术指标 (均线、MACD、RSI)
- 新闻资讯 (相关新闻摘要)
- 操作建议 (短线/中线/风险提示)

### 4. 消息推送模块 (Message Pusher)

**职责**: 将报告推送到飞书

**支持的消息类型**:
- 文本消息 (Markdown 格式)
- 图片消息 (图表文件)
- 富文本消息 (图文混排)

**推送渠道**:
- 飞书企业应用 (推荐)
- 飞书群机器人 Webhook

### 5. 系统自检模块 (System Checker)

**职责**: 确保系统稳定运行

**检查项**:
- Token 有效性
- Python 依赖完整性
- 图表生成能力
- 网络连通性
- 目录结构完整性

## 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 调度框架 | OpenClaw | 专为 AI Agent 设计，支持 Cron |
| 编程语言 | Python 3.7+ | 丰富的金融/数据生态 |
| 数据获取 | requests | 简单高效 |
| 数据处理 | pandas | 标准工具 |
| 图表生成 | matplotlib | 灵活、可定制 |
| 消息推送 | 飞书 API | 国内常用，支持图文 |

## 性能优化

### 1. Token 优化
- 直接执行 Python 脚本，避免 AI 推理
- 单次晨报 Token 消耗从 3000 降至 0

### 2. 缓存策略
- 数据缓存 5 分钟
- Token 缓存 2 小时
- 图表按需生成

### 3. 并发处理
- 数据收集并行化
- 图表生成异步化

## 安全设计

### 1. 敏感信息保护
- 使用 .env 文件存储密钥
- .gitignore 排除敏感文件
- 日志脱敏处理

### 2. 访问控制
- Token 定期刷新
- API 调用频率限制
- 错误信息不暴露内部细节
